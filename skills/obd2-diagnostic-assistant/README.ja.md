# OBD-II Diagnostic Assistant(OBD-II診断アシスタント)

[English](./README.md) | **日本語**

OBD-II故障診断コード(DTC)、フリーズフレームデータ、ライブセンサーデータを解釈し、スキャンツールの出力を構造化された診断計画に変えるAIスキルです。

## 説明

OBD-II診断アシスタントは、「チェックエンジンランプ(警告灯)が点いた」状態から、可能性の高い原因と検証テストの優先順位リストへ導きます。各コードが実際に何を監視しているのかを説明し、根本原因と二次的な影響を区別し、ライブデータ(燃料補正、O2センサーの挙動、エアフロメーターの値)の読み方を支援します。コードに書かれた部品をただ交換するのではなく、**故障の原因そのもの**を直すための道具です。

**対象:** スキャンツールを持つDIYユーザー、プロの整備士、警告灯の緊急度を知りたいすべてのドライバー

## できること

- 標準化されたDTC(P0xxx/P2xxx/P3xxxのパワートレイン系、B/C/U系)を平易な言葉で解説:何を監視し、どのようにテストが実行され、何がコードを記録させるのか
- メーカー固有コード(P1xxx)の解釈(メーカーごとに定義が異なる旨の注意付き)
- 複数コードが同時に出た場合の因果関係の分析(例:失火コード+リーンコード → バキュームリークの可能性)
- フリーズフレームデータから故障発生時の運転条件を再構築
- ライブデータ診断のガイド:短期/長期燃料補正、O2センサーの切り替わり、エアフロ計測値(g/s)と期待値の比較、水温の妥当性、EVAPモニタの状態
- レディネスモニタ(モニタ完了状態)の解説と、コード消去後に検査に合格するための条件
- 「そのまま走行可」から「レッカー推奨」までの緊急度判定
- 部品を買う前に診断を確定させるためのテストの選定と実施順序の提案
- OBD-IIプロトコル、コネクタピン配置の基礎、スキャナーの性能差(コードリーダー vs 双方向スキャンツール)の解説

## プロンプト例

```text
2015年式ホンダ シビックでP0420が出ました。これは必ず触媒の故障を
意味しますか?交換する前に何をテストすべきですか?
```

```text
V6エンジンでP0171とP0174が同時に出ています。両バンクがリーンという
のはどういう意味で、どこから点検を始めるべきですか?
```

```text
P0300(ランダム失火)のフリーズフレームデータです:[データを貼り付け]。
コードが記録されたとき、エンジンはどういう状態でしたか?
```

```text
長期燃料補正がアイドリングで+18%、巡航では+3%まで下がります。
このパターンは何を示唆していますか?
```

```text
コードを消去しましたが、検査で「モニタ未完了」と言われました。
2018年式トヨタ カムリのドライブサイクルの完了方法を教えてください。
```

## 推奨ワークフロー

1. **ペンディングコードを含め、すべてのコードを読み取る。** 最初の1つだけでなく全コードを報告してください — 組み合わせが重要です。
2. **消去する前にフリーズフレームを記録する。** コードを消すと証拠も消えます。
3. **車両情報を共有する。** 年式、メーカー、車種、エンジン、走行距離、最近の修理歴、現在の車両の挙動。
4. **因果分析を求める。** どのコードが根本原因で、どれが症状なのか。
5. **推奨テストを安い順に実施する。** 部品交換の前に目視点検とライブデータ。
6. **修理を確認する。** コード消去 → 該当ドライブサイクルの完了 → モニタが実行され合格することを確認。

## 制限事項

- **コードは「失敗したテスト」を示すもので、「壊れた部品」を示すものではありません。** P0420は触媒効率テストの不合格を意味し、原因はセンサー、排気漏れ、触媒本体のいずれもあり得ます。アシスタントは確率で推論し、物理的なテストが確定させます。
- **メーカー固有コード(P1xxx、大半のB/C/Uコード)はメーカーごとに異なります。** メーカー資料での確認が必要です。
- **スキャナーの画面は読めません。** コードとデータを正確に転記してください。1文字の誤記が誤った分析につながります。
- **ライブデータの解釈はデータの質に依存します。** 安価なELM327互換品は更新が遅い・不正確な場合があります。
- **OBD-II以前の車両**(米国では1996年より前など。日本車の国内仕様は義務化時期が異なります)は別のアプローチが必要です。
